⚙️ Compliance Engine (Eval Key)

📊
암호문 집계 복호 없이 누적액/카운트 계산
🚨
한도 트리거 일/월 한도 초과 시 플래그
📈
리스크 스코어 가중합 기반 이상거래 점수
📋
정기 보고 집계 통계만 복호 (원문 비공개)

🔐 증거 능력 확보

1 On-chain: audit_log_root (머클루트만 앵커링)
2 Off-chain: WORM 저장소에 암호문 저장 (변경불가)
3 감사 시: 온체인 root ↔ 오프체인 암호문 일치성 증명
💡 핵심: 평소엔 완전 비공개, 규제 트리거 시에만 최소 정보 선택 공개

A5 Issuer Controls

Issuer 통제: Freeze/Clawback/Blacklist

🔒

Freeze (동결)

특정 계정의 출금 차단

  • 잔고 금액은 비공개 유지
  • Unshield(출금) 시점에 권한 검증
  • 동결 상태는 암호화 플래그로 관리
🚫

Blacklist (차단)

제재 대상 입출금 원천 차단

  • Shield(입금) 시점 KYC ID 검증
  • Sanction list와 실시간 매칭
  • 차단 시 즉시 트리거 발생
↩️

Clawback (회수)

법원 명령 시 자금 회수

  • Threshold KMS t-of-n 승인 필요
  • 해당 TX만 선택적 복호
  • 회수 후 감사 로그 영구 기록
프라이버시 풀 + Issuer 통제 양립 구조
Shield 입구 KYC 검증 + Blacklist 체크
→
프라이버시 풀 내부 거래 비공개
→
Unshield 출구 Freeze 체크 + 출금 승인
🎯 핵심: 풀 내부는 프라이빗, 입출구(Shield/Unshield)에서 Issuer 통제 적용
