03 Problem

블록체인 결제의 딜레마

문제 A

투명한 블록체인 = 정보 노출

❌ 모든 거래가 실시간 공개
❌ 경쟁사가 거래 패턴 분석 가능
❌ 기업 전략, 급여 정보 유출
기업이 사용 거부
VS
문제 B

프라이버시 코인 = 규제 불가

❌ AML/KYC 감사 불가능
❌ Tornado Cash → OFAC 제재
❌ Zcash, Monero → 거래소 상장폐지
규제기관이 금지
🔒
핵심 딜레마
완전 투명 → 기업이 안 씀  |  완전 비공개 → 규제기관이 금지
→ 프라이버시 + 선택적 감사가 유일한 해답

06 Core Technology

ZK + FHE 역할 분리 아키텍처

⚡

ZK-SNARK (Noir/Barretenberg)

전송 검증
목적: 저비용 전송 + 정책/분기 준수 증명
회로Plonk + UltraHonk
증명~12초 (서버)
검증< 1ms (온체인)
잔액 검증 정책 준수 if/else 분기
🔐

FHE16 (RLWE)

감사 엔진
목적: 암호문 상태에서 규제 신호 계산
파라미터N=1024, Q=167,772,161
Ciphertext1056 fields (32 c0 + 1024 c1)
보안128-bit 양자내성
누적 집계 한도 트리거 감사 태그
1 ZK로 전송 유효성 + 정책 준수 증명
→
2 RLWE로 Audit Tag 암호화 기록
→
3 Eval Key로 누적/트리거 계산
→
4 Threshold 복호 (트리거 시)
💡 핵심: ZK는 싸고 빠른 전송, FHE는 프라이빗 규제 엔진 → 역할 분리로 비용 최적화
