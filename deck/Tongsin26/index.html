Part 2

MPC의 통신 한계

왜 기존 암호 프로토콜은 블록체인에서 작동하지 않는가

MPC 개요

다자간 연산(MPC)의 두 가지 방식

1. Garbled Circuits (GC)

회로를 "난독화"하여 전송, 상대방이 평가

  • 통신량: O(|회로|) — 회로 크기에 비례
  • 라운드: 상수 (2-3회)
  • 문제: 회로 재사용 불가, 2자 전용

2. Secret Sharing (SS)

데이터를 조각으로 나눠 분산 연산

  • 통신량: O(1) per gate
  • 라운드: O(곱셈 수) — 핵심 문제!
  • 문제: 매 곱셈마다 통신 필요

공통된 근본 문제

두 방식 모두 참여자 간 통신이 필수입니다. 블록체인처럼 전 세계에 분산된 시스템에서는 통신 지연이 성능을 지배합니다.

100ms+

서울 ↔ 미국 간 최소 통신 지연

광섬유에서 빛은 진공의 약 67% 속도로 이동합니다.
11,000km를 왕복하는 데 물리적 최소 110ms가 필요합니다.
이것은 기술 발전으로 줄일 수 없는 물리 법칙의 한계입니다.

  • 검증자: 합의 타임아웃
  • 블록체인: 처리량 붕괴

결론: SS-MPC는 글로벌 블록체인에서 근본적으로 불가능

Garbled Circuits

GC-MPC도 해결책이 아닌 이유

Garbled Circuits는 라운드 수는 적지만, 다른 문제가 있습니다:

1

통신량 폭발

256비트 곱셈 회로 ≈ 500KB. 복잡한 컨트랙트 = GB 단위 전송

2

일회용 회로

같은 회로 재사용 불가. 매 트랜잭션마다 새로 생성/전송

3

2자간 프로토콜

N명의 검증자로 확장 시 복잡도 급증

근본적 한계

MPC는 "알려진 참여자 간의 대화형 프로토콜"을 위해 설계되었습니다.

블록체인은 "불특정 다수 검증자의 비대화형 합의"를 요구합니다.

패러다임 자체가 맞지 않습니다.

Part 3

FHE: 통신 없는 암호 연산

완전 동형 암호로 패러다임을 전환하다

FHE 개요

완전 동형 암호(FHE)란?

암호화된 상태에서 연산이 가능한 암호 체계

1

사용자: 데이터 암호화

Enc(x) — 비밀키로 암호화

2

서버: 암호문으로 연산

f(Enc(x)) = Enc(f(x))

3

사용자: 결과 복호화

Dec(Enc(f(x))) = f(x)

통신 관점에서의 핵심

  • 연산 중 통신 불필요
  • 서버는 암호문만으로 독립 연산
  • 여러 서버가 동시에 같은 연산 가능
  • 결과는 항상 동일 (결정적)

블록체인에 완벽한 이유

각 검증자가 독립적으로, 통신 없이 동일한 연산을 수행하고, 동일한 결과에 도달합니다.

비교

MPC vs FHE: 통신 복잡도

항목 Secret Sharing MPC Garbled Circuits FHE
연산 중 통신 매 곱셈마다 필요 초기 전송 필요 불필요
라운드 복잡도 O(곱셈 깊이) O(1) O(0)
통신량 O(n) per gate O(|회로|) O(1) 입출력만
다자간 확장 통신량 증가 복잡도 급증 영향 없음
글로벌 블록체인 불가능 비실용적 가능

결론

FHE는 유일하게 통신 라운드가 0인 암호 연산 방식입니다. 이것이 글로벌 블록체인에서 FHE만이 작동하는 이유입니다.

FHE16: 고속 부트스트래핑

IEEE Access 2025 · ePrint 2024/1916

핵심 혁신

16비트 정수 연산으로 모든 것을 대체

기존 방식 (TFHE-rs)

64비트 FFT (Fast Fourier Transform) 사용

  • 부동소수점 반올림 오류
  • 플랫폼마다 다른 결과
  • 복잡한 명령어 시퀀스

FHE16 방식

16비트 NTT (Number Theoretic Transform)

  • 순수 정수 연산만 사용
  • 모든 플랫폼에서 동일한 결과
  • SIMD 최적화에 유리

왜 16비트인가?

Intel vpmulhw 명령어:

16bit × 16bit → 상위 16bit 추출

단 1 CPU 사이클에 실행

512비트 AVX512 레지스터에서 32개 연산 동시 수행

합성 모듈러스
q = p₁ · p₂
p₁, p₂는 16비트 소수
통신 관점

결정적 연산: 합의의 핵심

TFHE-rs (FFT 기반)

부동소수점 반올림이 다음에 따라 달라짐:

  • CPU 아키텍처 (x86 vs ARM)
  • 컴파일러 버전 및 최적화
  • 명령어 스케줄링

→ 검증자마다 다른 결과
→ 합의 실패

FHE16 (정수 기반)

모든 연산이 정수 모듈러 산술

  • 같은 입력 → 같은 출력
  • 플랫폼, 컴파일러 무관
  • 비트 단위 완벽한 재현성

→ 모든 검증자가 동일한 결과
→ 합의 성공

Threshold FHE: 모든 참여자가 동일한 암호문을 생성해야 복호화 성공

요약

핵심 논지

1

블록체인은 전 세계 분산 시스템

검증자가 한국, 미국, 유럽 등에 분산 — 물리적 통신 지연 불가피

2

MPC는 통신에 의존

Secret Sharing: 곱셈마다 라운드 필요 → 54초/트랜잭션
Garbled Circuits: 회로 전송 필요 → 대역폭 문제

3

물리 법칙이 MPC를 막음

서울-버지니아 RTT ~180ms는 광속의 한계. 기술 발전으로 개선 불가.

4

FHE는 통신이 필요 없음

각 검증자가 독립 연산 → 라운드 복잡도 O(0)

5

FHE16이 FHE를 실용화

2.89ms 부트스트래핑 + 결정적 연산 → 글로벌 블록체인에서 작동
